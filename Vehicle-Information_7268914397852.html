VDC : Vehicle Information

Created by Stephan Oomen on 27 dec., 2021

base_url:             /api/v1/vehicles

Information

Retrieves  vehicle information based on the subscription 

URL

{base_url}/information

Method (Verb)

POST

Headers

Accept: application/json

Content-Type: application/json

OR

Accept: text/xml
Content-Type: text/xml

Parameters

  • Reference (optional)

  • License Plate (required)

Request body

JSON format

 

{
   "reference":string,
   "licensePlate":string
}

XML format

<?xml version="1.0"?>
<VehicleRequest>
  <Reference>string</Reference>
  <LicensePlate>string</LicensePlate>
</VehicleRequest>

Success Responses

200: JSON format

200: XML format

Invalid Responses

400: JSON format

{
  "LicensePlate": [
    "The LicensePlate field is required."
  ]
}

400: XML format

<Error>
   <LicensePlate>The LicensePlate field is required.</LicensePlate>
</Error>

Response format explanation

The response object returns the following object:

Property

In JSON

In XML

Description

Response Data

“responseData”

<ResponseData>

Array of responses containing a Vehicle Response Item

Vehicle Response Item

{}

<VehicleResponseItem>

Item holder containg response information

 

The Response Data is an array of responses since there can be multiple actions done on a single endpoint.

Vehicle Response Item

The Vehicle Response Item contains the following properties 

Property

In JSON

In XML

Description

Succeeded

“succeeded”

<Succeeded>

Whether this response type succeeded

Name

“name”

<Name>

What kind of result of the request is. Can be “information”, “registration”

Type

“type”

<Type>

Entity Type, can be “vehicle”

Data

“data”

<Data>

Set of properties returned based on the request type and name

Data -> Item

“{key}”:”{value}”

<Item Key="{key}"> <![CDATA[{key}]]>

</Item>

Object property of the type. On “information” is the information based on the type. On “registration” is the data returned to relate to the registration.

 

As mentioned, the Item is information or registration data from the kind of request.

For information this can be very dynamic, it depends what kind of subscription you have and which fields that include.

On registration we return an “id” that is the idenficiation for the registration.

Examples

URL:                       /api/v1/vehicles/information

Headers
Content-Type:   application/json
Accept:                 text/xml
Authorization:   Basic <basic>
Content-Length: 49

Body

{
  "reference": "",
  "licensePlate": "99XXXX"
}
