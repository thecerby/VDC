VDC : Batchroutines

Created by Stephan Oomen, last modified on 06 apr., 2023

On the VDC-PORTALS server, there is an application called "Batchroutines". It is basically an scheduler and calling out an service at that time. Make sure that this application is running on this server.

In this application, it calls the following tasks:

  • Groene kaart

  • BPM Belastingaangifte

  • BTW Controle (1 valid, 1 invalid check)

  • ORB check

In all cases it calls out to the DataConnectionWcf webservice

Groene kaart

This routine sends information to Maral (vehicle information and documents)

The following APIs are being used to send it to Maral

  • http://v1.vdcnederland.eu/wsconnect/wsconnection.asmx sending customer data

  • http://wservice.vdcnederland.eu/vdc_loadorderdata.asmx sending the actual document.

BPM Belastingaangifte

This routine prepares vehicles that has been asked for BPM return when an vehicle gets exported.
It check only "aangiftes" when it is still open, sets the dates and assign the vehicles that are sold in that period. From there, the RegistratieXpertVdaInterface takes over the rest (this is done by VDC backoffice)
This is available for specific subscribers.

This routine is no longer in use due to changes in RE2.0.

BTW controle

Every day we check if the VAT numbers are still valid, every week we check if the invalid VAT numbers are now valid. This goes through the whole tblKlant table that are not deleted.

ORB check

For the subscribers containing an A2SP account number, we do a nightly check if the vehicle is still on the ORB of the subscriber. If not, the database will be updated.
