base_url:             /api/v1/bca/vehicles

Registration

Registers the vehicle as ‘new’ in the RegistratieXpert.
The following data will be enriched in the RegistratieXpert outside the supplied data upon registration;

  • Reliability of odometer (NAP check)

  • Ovi vehicle information

  • Vehicle information document (VoertuigGegevens)

  • Certificate of Origin document (CvO)

  • Euro Certificate (Euro Certificaat)

  • Checks in RDW if the vehicle is on company stock (marks as in stock yes/no)

The registration is accepted as is, if – for example – the reportCode is invalid, the vehicle is still created along with the specified buyer (if applicable) but without the CvO.

 

If the transaction is expected to fail, please contact VDC for additional requirements.

URL

{base_url}/registration

Method (Verb)

POST

  • Attendance Code2 (required when Document Number is not empty)

Response format explanation

The response object returns the following object:

Property

In JSON

Description

Response Data

“responseData”

Array of responses containing a Vehicle Response Item

Vehicle Response Item

{}

Item holder containing response information

 The Response Data is an array of responses since there can be multiple actions done on a single endpoint.

Vehicle Response Item

The Vehicle Response Item contains the following properties 

Property

In JSON

Description

Succeeded

“succeeded”

Whether this response type succeeded

Name

“name”

What kind of result of the request is. Can be “information”, “registration”

Type

“type”

Entity Type, can be “vehicle”

Data

“data”

Set of properties returned based on the request type and name

Data -> Item

“{key}”:”{value}”

Object property of the type. On “information” is the information based on the type. On “registration” is the data returned to relate to the registration.

As mentioned, the Item is information or registration data from the kind of request.

For information this can be very dynamic, it depends what kind of subscription you have and which fields that include.

On registration we return an “id” that is the identification for the registration
