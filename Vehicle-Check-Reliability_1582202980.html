VDC : Vehicle Check Reliability

Created by Kavitha Thangappan, last modified on 11 nov., 2021

Customer who wants to check reliability submits request to this endpoint.

System calls A2SPService - ControleerTellerStand method by passing LicensePlate , CounterPostion , Counter Type details as input parameters.

If the result is not successful system throws error message as NapResultUnKnown. Otherwise system returns NapResult details.

For using this api, you need to be authorized.

Authentication

Please refer to this page. Authentication APIs - VDC - Confluence (atlassian.net)

Authorization

This requires api permission Pages_Vehicles and feature permission RdwRequestNapFeature

 

Property

Value

Endpoint

​/api​/services​/app​/VehicleCounter/CheckReliability

Method

POST

Authorization

Token {token from Authorized User}

Headers

Accept: application/json 

Content-Type: application/json 

Field Parameters

  • licensePlate- String- (required) 

  • counterPosition- Integer- (required) 

  • CounterType- String- (required) 

 

Input

{
  "licensePlate": "29XSR5",
  "counterPosition": 10000,
  "counterType": "K"
}
