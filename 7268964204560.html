VDC : Deploy RegistratieXpert changes to [environment]

Created by Stephan Oomen, last modified on 28 mei, 2024

When you want to make changes to the RegistratieXpert, you need to submit a Pull Request and the changes needs to be pushed towards the dev branch of RegistratieXpert . When that is done, please follow below instrunctions;

 Instructions

For TST environment, after the PR is completed to dev the pipeline automates the way of deploying the changes to TST. Check the following release pipelines for the status;

  • registratiexpert-test-api

  • registratiexpert-test-ui

For the RegistratieXpertServices application, you check

  • registratiexpert-services

If you need to rollout to UAT and PRD, create a PR from dev to master and complete the PR via rebase flow (NEVER DELETE THE dev branch)

The pipelines will be run for the master branch. Wait for it to complete and create a release for

  • registratiexpert-api (if there are any API changes)

  • registratiexpert-ui (if there are any UI changes)

For the RegistratieXpertServices application, you need to deploy for UAT, PRD etc..

  • registratiexpert-services

You need approval from Jeroen or Rick in order to deploy to production! (they will be informed via email)

Database changes

if there are any database changes, you need to connect to the SQL server VDC-SQL-TEST for TST and UAT.
For production you need to connect to VDC-WEB-PRD. Credentials are in access.mplus.nl

There on the desktop there is a shortcut named

TST => RegistratieXpert.Migrator - TST
UAT => RegistratieXpert.Migrator - UAT
PRD => RegistratieXpert.Migrator - PRD

Start this application and type 'Y' when you want to proceed running the migration scripts.

If there is an connection issue, this could be due to the hosts file not correctly being read. Try changing cloudapp.net to registratiexpert.net in the connectionstrings of the migrator.
