VDC : Could not create SSL/TLS secure channel.

Created by Stephan Oomen on 04 jul., 2022

It can happen that customers of VDC gets the following error;

This can happen based on the following things;

  1. RDW certificate is expired

  2. Connection to A2SP is not working correctly

The below instructions will check and resolve RDW certificate is expired

You’ll need to know the following things;

  • What is the tenant name and/or tenant Id

 Instructions

Getting the thumbprint of the current certificate;

  1. Log into http://registratiexpert.nl and go to tenants

  2. Search for the tenant and click on Acties => Details

  3. Navigate to tab RDW and copy the value RdwCertificaatThumbprint

Searching the certificate

  1. Connect via RDP on VDC-PRD-WEB which is found in access.mplus.nl

  2. In the machine, open up mmc.exe and add the snap-in Certificates for local computer

  3. Rightclick on “Certificates” and click on Find Certificates

  4. Fill in the thumbprint value in the Contains field and look in field SHA1 Hash

  5. Check if the expiration date is expired.

You can double click the certificate and it will show as expired if that is the case.

The certificate is expired? Follow below;

Now you know the certificate is expired and you need to install the new one, connect with VDC to get the new certificate.

Install the certificate

Copy the pfx file towards the webserver

  1. and double click the file.

  2. Choose Local Machine and click ‘Next’

  3. Click next

  4. Specify the password and check “Mark this key as exportable”

  5. Click next and install.

Password is invalid? Double-check the characters, it could be that there is a mismatch, otherwise connect with VDC

  1. Follow the steps of Searching the certificate and look for the certificate by looking into the Personal => Certificates folder.

  2. Open up the certificate and go to Details

  3. Search for field Thumbprint and copy the value.

  4. Put this value into notepad++, remove alle whitespaces and a ‘hidden’ character at the start.

  5. Place the new thumbprint value into the details section and save the changes.

  6. Let VDC ask to check the application again if it is working.
