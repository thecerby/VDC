VDC : Detect Check

Created by Stephan Oomen, last modified on 12 okt., 2023

base_url:             /api/v1/vehicles/insurance

Checks the user if it is eligible to provide an insurance

URL

{base_url}/api/v1/vehicles/insurance/check

Method (Verb)

POST

Headers

Accept: application/json

Content-Type: application/json

OR

Accept: text/xml
Content-Type: text/xml

Request body

JSON format

 

{
  "achternaam": "string",
  "voornaam": "string",
  "initialen": "string",
  "tussenvoegsel": "string, optional",
  "geboortedatum": " dd-MM-yyyy",
  "postcode": "string, specify when it is for private use",
  "huisnummer": "string, specify when it is for private use",
  "straat": "string, specify when it is for private use",
  "plaats": "string, specify when it is for private use",
  "land": "string, specify when it is for private use"
}

Success Responses

200: JSON format

{
  "isAccepted": true,
  "score": 11,
  "alertId": 66665,
  "argumentation": "GROEN: Geen match op sanctie- of PEP-lijst",
  "indicators": [],
  "logVariables": [],
  "warnings": []
}

 

Response format explanation

 The response object returns the following object:

Property

In JSON

Description

Is Accepted

“isAccepted”

Determines if the check has been accepted, this Boolean defines  if you can provide the insurance

Argumentation

“argumentation”

Detailed response on the check

Score

“score”

Defines the score of the check

Indicators

“indicatiors”

List of possible indications on the check.
