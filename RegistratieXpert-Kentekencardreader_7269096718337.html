VDC : RegistratieXpert Kentekencardreader

Created by Stephan Oomen on 28 jul., 2023

Vinacles and BCA has a copy of the kentekencardreader which allows the user to read license plate cards and import them into RegistratieXpert. When you insert the license plate card, the app will read the information and show a representation of the card in the eVR tab.

For this to work, you’ll need the following device;

Connect the cardreader with USB and start the application.

Authorizing the kentekencardreader

When you want to add the vehicle to the RegistratieXpert you have to log in. To do that you need to login as a tenant in the website app. There are 2 ways of logging in;

  • Tenant login, the URL is known for the tenant, e.g. BCA will have http://bca.registratiexpert.nl

  • Host login, the URL is http://registratiexpert.nl , you login with admin credentials and go to “Tenants” and click on “log in as tenant”.

When done, you can connect the kentekencardreader with a device code.

In the website, top-right corner you click on the profile:

Then click on “device login”. A screen will show where you can enter a device code.

Now in the kentekencardreader, in the “RegistratieXpert” tab, click on “Inloggen”

A dialog will show with a code, copy this and fill it in the dialog you have opened in the website.

If all goes well, a confirmation screen will show if you want to allow access to the cardreader

Confirm the dialog and the kentekencardreader will automatically close the authorization dialog and show the logged in user.

Adding a vehicle to the RegistratieXpert

Now, get a licenseplate card and push it into the cardreader. The application automatically recognizes it and show a representation of the card in the eVR tab.

To add it to the RegistratieXpert, simply go to “RegistratieXpert” and you’ll see the licensePlate showing in the screen.

If you want, add the mileage, document numbers and attendance codes.

And click on “Voertuig toevoegen”. The application will connect to the RegistratieXpert through an API and check if the vehicle with the licensePlate exists.

If the vehicle exists, a question will be raised if you want to continue and add it to the existing vehicle.

Finally the following information is sent to the RegistratieXpert

  • A vehicle is created (when not existing)

  • Mileage will be checked (if supplied)

  • Card information will be sent and update the vehicle

  • The screens shown in eVR will be sent and saved as documents and attached to the vehicle.

When the operation is completed. An alert is shown to the user that the vehicle is either added or updated.

Technical notes

The application is based on the existing code found in Github: https://github.com/eVRMTV/eVR

Someone shared a blog post talking more in detail about this; https://blog.kenteken.tv/2020/03/12/er-staat-meer-data-op-je-kentekencard-dan-je-denkt/

The libraries used are for checking the authorization and reading the information from the card.

Debugging

To enable debugging, you need to set the switches in the config file to Verbose

<switches>
      <add name="CardReader" value="Verbose" />
      <add name="Registration" value="Verbose" />
      <add name="MTVCardReader" value="Verbose" />
      <add name="ElementaryFile" value="Verbose" />
    </switches>

It will create a Diagnostics.log file next to the executable that will show when the card is added, monitoring is connected to the USB device and authorization is checked.

If for some reason the card is not accepted, this is a way to check if the checksums are correct.
Detailed information about the authorization and checksums can be found here:
https://github.com/eVRMTV/eVR/blob/master/doc/DA12%20-%20Kentekencard%20Uitleesdocumentatie%20v2%201%201.pdf

Making the app available

C:\RegistratieExpert\Kentekencardreader\unsigned
Production: http://vdc.registratiexpert.nl/kentekencardreader/unsigned/[PRD]Kentekencardreader.zip
Test: http://vdc.registratiexpert.nl/kentekencardreader/unsigned/[TST]Kentekencardreader.zip

Versions

The difference between the 2 versions is the appSetting VdcApiClient:Url in the config file.

Values are;

Test: https://regxpert-api-test.azurewebsites.net/

Uat: https://regxpert-api-uat.azurewebsites.net/

Prd: https://vdc-api.registratiexpert.nl/
