VDC : Upload Entity Document

Created by Kavitha Thangappan on 11 nov., 2021

Customer who wants to upload existing entity document details sends request to this endpoint.

If the entity type equals vehicle , system fetches document details from vehicle document repository based on the document id provided as input parameter.

If the document is not available system throws user exception as DocumentDoesn'tExists

system fetches subdirectory value from contact person repository as identification number if the document id provided is available in the contact person's document list.

Otherwise it sets subdirectory as license plate from vehicle object.

System fetches document name from vehicle document repository.

If the document is not available system throws user exception as DocumentDoesn'tExists

System calls DocumentService - PrepareDocument by passing document Id ,vehicle Id also the subdirectory.

Then system calls DocumentService - UploadDocument by passing document which is from customer document repository and binary data provided in the input object.

if the upload is un successful system throws exception as DocumentUploadFailed.

If the entity type equals contact person, system fetches document details from contact person repository.

If the document is not available system throws user exception as DocumentDoesn'tExists

System calls DocumentService - PrepareDocument by passing document Id ,vehicle Id also the subdirectory.

Then system calls DocumentService - UploadDocument by passing document which is from customer document repository and binary data provided in the input object.

if the upload is un successful system throws exception as DocumentUploadFailed.

For using this api, you need to be authorized.
