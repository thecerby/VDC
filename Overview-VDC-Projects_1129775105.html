VDC : Overview VDC Projects

Created by Stephan Oomen on 26 mrt., 2021

RegistratieXpert

The "RegistratieXpert" (RX) is an application built in Silverlight. This is found in the "RegistratieXpert" repository.
Solution can be found in src/RegistratieXpert/ExportXpert.sln.

Be sure to open this in Visual Studio 2015, VS2017 does not support Silverlight.
Also, install the required components which can be found in tools/install

When deploying the Silverlight app, it must be built in "Release" mode.
! do not disable the certificate because this is required for drivers to be loaded with Silverlight.

RX Services

RX uses WCF services communicate, this lives in ExportXpert.Web. ExportXpertServices are the central APIs that is used by multiple applications, e.g.:

  1. RegistratieXpert

  2. KentekencardReader

  3. VWEServices a.k.a. BCA Services

A2SPServices

This is used for retrieving CvO for an vehicle

CrmServices

This is primary used in RegistratieXpert for the current subscriber and checking the VAT number.

DocumentService

Used for CRUD'ing vehicle documents

EmailServices

Used for sending prepared emails in the tblMailRequests

RdwServices

Includes all RDW related services. This calls either the Portals or de A2SP XML API directly.
And saves it in the database.
This is based on the data configured in the tblAbonnee table.

VehicleService

Able to retrieve and update the vehicle information in the database.

BCA Services -> VWEServices

The BCA Service is being updated to VWE webservice style. The service has been moved away from the RegistratieXpert Repository to it's own - VWEServices - repository. This service is now configured to deploy using the VSTS Build and release tooling.

Services

The VWE Services contains the following services:

  • ATD (Aanvraag tijdelijk documentnummer)

  • CB (Controle bedrijfsvoorraad)

  • OKR (Online kilometer registratie)

  • ORB (Online registratie bedrijfsvoorraad)

Before an vehicle can be sent to ORB, the vehicle needs to be created in RX first. This can be done by calling the OKR webservice, the DataXpert registration or by manually adding in inside RX.

Documentation per service is found in the repository.

DataConnectionWCF

This service is the backend for the Portal applications, batchroutines and the RegistratieXpertVdaInterface.

  • Polling for the next request found in tblPortalAanvraag

  • Belasting

  • Groene kaart registration

  • Voertuig export (Vies)

KentekenCardReader

Used for reading kentekencards. run the application, wire up the USB cardreader (found in Stephan's desk) and login using RX credentials.

RDW portals

Here are the RDW portals, currently OREH and ORED can be no longer used. ORB is used when using A2SP XML API.

VDC.Services

This is also called the DataXpert, it contains the possibility to request vehicle information and vehicle registration (combined). The ASR data check has been added here too.

The service has been moved away from the RegistratieXpert Repository to it's own - VDCServices - repository. This service is now configured to deploy using the VSTS Build and release tooling.

Setting up the environment.

When you want to fully run the RX application on your local machine, you need to run the following applications in debug mode.

  • ExportXpert.Web

  • ExportXpertServices

  • DataConnectionWcf (if you want to go through the portal procedure)

  • The portal you want to test

Also, make sure to connect the SQL database and install the required certificates.
